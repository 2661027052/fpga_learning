# 建立时间与保持时间

## 问题

> 什么是建立时间和保持时间？写出系统级公式。违反后如何修复？为什么 Hold 违例与频率无关？

> 难度: ★★ | 频率: 必问 | 层级: L2

## 结论

建立时间(Setup) = 时钟沿到来前数据必须稳定的最小时间；保持时间(Hold) = 时钟沿到来后数据必须保持的最小时间。违反 Setup → 降频/加流水线；违反 Hold → 加延迟。

## 基本定义

```
时钟沿:     ↑
数据:   ────┐    ┌────
           │<──>│<──>
           Setup Hold
```

| 参数 | 含义 | 检查时刻 |
|------|------|---------|
| Tsetup | 数据在时钟沿**前**需要稳定 | 时钟沿之前 |
| Thold | 数据在时钟沿**后**需要保持 | 时钟沿之后 |

## 系统级公式

- **Setup**: Tco + Tlogic + Tsetup ≤ Tclk - Tskew
- **Hold**: Tco + Tlogic ≥ Thold + Tskew

## 关键考点

1. Setup 与路径**最大延迟**有关（最慢工艺角）
2. Hold 与路径**最小延迟**有关（最快工艺角）
3. Hold 违例与时钟频率**无关**——降频不能修复 Hold
4. 符号解释：Tco=触发器输出延迟, Tlogic=组合逻辑延迟, Tskew=时钟偏斜
