# 多比特 CDC 跨时钟域

## 问题

> 多比特信号跨时钟域有哪几种方法？各自的优缺点？为什么不能直接同步多比特总线？

> 难度: ★★ | 频率: 高频 | 层级: L2

## 结论

多比特 CDC 有三种方法：握手协议（少量低速）、异步 FIFO（高速大量）、DMUX 同步（数据+控制信号）。最通用的是异步 FIFO。

## 为什么不能直接同步多比特？

不同 bit 布线延迟不同 → 时钟沿采样时各 bit 可能处于不同转换阶段 → 采到垃圾组合值（如 1100→1010 时采到 1110）。

## 方法对比

| 方案 | 吞吐量 | 延迟 | 复杂度 | 适用场景 |
|------|:-----:|:----:|:-----:|---------|
| 握手协议 | 低 | 高(2-3周期) | 低 | 少量数据/控制信号 |
| DMUX同步 | 中 | 中 | 低 | 数据变化不频繁 |
| 异步FIFO | 高 | 低(1拍) | 高 | 大量数据/高带宽/最推荐 |

## 握手协议核心代码

```verilog
// 发端: req_a 表示数据就绪
if (data_valid && !req_a) begin
    data_hold <= data_in;
    req_a <= 1'b1;
end else if (ack_a_sync)  // 收到应答
    req_a <= 1'b0;

// 收端: 同步 req, 收到后锁存数据, 发出 ack
if (req_b_sync2) begin
    data_out <= data_hold;
    ack_b <= 1'b1;
end else
    ack_b <= 1'b0;
```

## 工程建议

- 数据宽度 ≤ 8bit 且速率低 → 握手
- 数据宽度 ≥ 8bit 且速率高 → 异步 FIFO
- 能用 FIFO 的地方尽量用 FIFO（验证最成熟）
