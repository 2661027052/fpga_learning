# 手撕代码：脉冲同步器

## 问题

> 设计一个脉冲同步器，将单脉冲从时钟域 A 同步到 B。用 Toggle 翻转法，写完整 Verilog。为什么不能直接用两级同步器同步脉冲？

> 难度: ★★ | 频率: 高频 | 层级: L2

## 结论

脉冲→电平翻转→两级同步电平→边沿检测→恢复脉冲。Toggle 翻转法是单脉冲跨时钟域标准答案。

## 完整代码

```verilog
module pulse_synchronizer (
    input  wire clk_a, rst_a_n, pulse_a,
    input  wire clk_b, rst_b_n,
    output wire pulse_b
);
    // 发送端：脉冲 → 电平翻转
    reg toggle_a;
    always @(posedge clk_a or negedge rst_a_n) begin
        if (!rst_a_n) toggle_a <= 1'b0;
        else if (pulse_a) toggle_a <= ~toggle_a;
    end

    // 接收端：两级同步
    reg sync_b1, sync_b2;
    always @(posedge clk_b or negedge rst_b_n) begin
        if (!rst_b_n) begin sync_b1 <= 0; sync_b2 <= 0; end
        else begin sync_b1 <= toggle_a; sync_b2 <= sync_b1; end
    end

    // 边沿检测恢复脉冲
    reg sync_b3;
    always @(posedge clk_b or negedge rst_b_n) begin
        if (!rst_b_n) sync_b3 <= 0;
        else sync_b3 <= sync_b2;
    end
    assign pulse_b = sync_b2 ^ sync_b3;
endmodule
```

## 为什么不能直接同步脉冲？

| 场景 | 问题 |
|------|------|
| 快到慢 | 脉冲宽度 < 慢时钟周期，可能采不到 |
| 慢到快 | 脉冲宽度 > 快时钟周期，可能被多次采样 |

Toggle 解法本质：将"事件"编码为"状态"（电平），同步状态后再解码为"事件"。

## 面试追问

Q: 最小脉冲间隔？
A: 接收端需 ≥ 2 个 clk_b 周期检测一次翻转。两个 pulse_a 间隔太短会被合并为一个脉冲。
