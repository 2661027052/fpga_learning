# 时序违例修复方法

## 问题

> 什么是 Setup Violation 和 Hold Violation？各自的修复方法有哪些？两者的根本区别是什么？

> 难度: ★★ | 频率: 高频 | 层级: L2

## 结论

Setup Violation = 数据走太慢（来不及）；Hold Violation = 数据走太快（被冲掉）。Setup 可降频/加流水线修复，Hold 只能加延迟修复且**与频率无关**。

## 根本区别

| | Setup Violation | Hold Violation |
|------|:---:|:---:|
| 公式 | Tco+Tlogic+Tsetup > Tclk | Tco+Tlogic < Thold |
| 含义 | 数据来不及到达 | 数据被新数据冲掉 |
| 与频率 | 有关（降频可修复） | **无关**（降频无效）← 考点 |
| 工艺角 | 最慢 corner | 最快 corner |

## Setup 修复方法（按优先级）

1. **降频** ← 最简单，降低时钟频率
2. **加流水线** ← 将长组合路径拆成多级
3. **重写关键路径代码** ← 减少逻辑级数
4. **改变综合策略** ← retiming / 面积速度权衡

## Hold 修复方法

1. **加 delay buffer** ← 标准方法
2. **加 LUT** ← 用查找表延迟
3. **调整布局布线** ← 增加路径延迟

## 面试必答

"Hold 违例与时钟频率无关是经典考点——因为 Hold 检查的是数据是否被太快冲掉，降频只会让时钟周期变长，对 Hold 窗口无影响。修复 Setup 可以降频，修复 Hold 只能加延迟。"
