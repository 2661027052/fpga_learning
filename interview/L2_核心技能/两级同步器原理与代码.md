# 两级同步器原理与代码

## 问题

> 两级同步器的工作原理？写出标准 Verilog 代码。为什么是两级不是一级？有哪些约束条件？

> 难度: ★★★ | 频率: 高频 | 层级: L2

## 结论

两级同步器是单比特跨时钟域的标准方案——两级触发器级联，给第一级一个完整时钟周期从亚稳态恢复。

## 标准 RTL

```verilog
module sync_2ff (
    input  wire clk_dst,     // 目标时钟
    input  wire rst_n,       // 异步复位（低有效）
    input  wire sig_async,   // 异步输入信号
    output reg  sig_synced   // 同步后输出
);
    reg sync_ff1;
    always @(posedge clk_dst or negedge rst_n) begin
        if (!rst_n) begin
            sync_ff1   <= 1'b0;
            sig_synced <= 1'b0;
        end else begin
            sync_ff1   <= sig_async;   // 第一级：可能亚稳态
            sig_synced <= sync_ff1;    // 第二级：已稳定
        end
    end
endmodule
```

## 为什么是两级？

| 级数 | MTBF(200MHz) | 结论 |
|------|-------------|------|
| 一级 | 几秒 | 不可接受 |
| 两级 | 数百年 | 标准方案 |
| 三级 | 数十亿年 | 过度设计 |

## 关键约束

1. 两级 FF 必须在同一个 always 块中（确保综合工具放相邻位置）
2. 两级之间不能插入组合逻辑（会破坏恢复时间）
3. 只能用于单比特信号（多比特总线必须用异步 FIFO 或握手）

## 常见误区

| 误区 | 真相 |
|------|------|
| 同步器消除亚稳态 | 不能消除，只能降概率 |
| 一级就够用 | 一级 MTBF 只有几秒 |
| 总线也可以打两拍 | 各 bit 延迟不同，数据可能错拍 |

## 边沿检测（加分）

```verilog
always @(posedge clk) begin
    sync1   <= sig_async;
    sync2   <= sync1;
    sync_d1 <= sync2;
end
wire pos_edge = sync2 & ~sync_d1;  // 上升沿
wire neg_edge = ~sync2 & sync_d1;  // 下降沿
```
