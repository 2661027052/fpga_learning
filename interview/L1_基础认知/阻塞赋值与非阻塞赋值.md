# 阻塞赋值与非阻塞赋值

## 问题

> 阻塞赋值 `=` 和非阻塞赋值 `<=` 的本质区别是什么？组合逻辑和时序逻辑各应该用哪种？

> 难度: ★ | 频率: 高频 | 层级: L1

## 结论

**黄金法则：组合用 `=`，时序用 `<=`**。

## 核心区别

| | 阻塞 `=` | 非阻塞 `<=` |
|------|---------|-----------|
| 执行顺序 | 立即，阻塞后续 | 并行，统一更新 |
| 硬件映射 | 组合逻辑 | 时序逻辑(DFF) |
| 仿真行为 | 类似C语言 | 硬件行为 |

## 关键示例

```verilog
// 时序逻辑：必须用 <=
always @(posedge clk) begin
    q1 <= d;      // q1 得到 d 的旧值
    q2 <= q1;     // q2 得到 q1 的旧值 → 移位寄存器
end

// 组合逻辑：必须用 =
always @(*) begin
    tmp = a + b;  // tmp 立即得到新值
    out = tmp + c;// out 得到 a+b+c
end
```

## 常见错误

时序逻辑中混用 `=` → 仿真与实际硬件行为不一致 → 仿真过、上板错。
