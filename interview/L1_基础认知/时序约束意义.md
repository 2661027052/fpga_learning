# 时序约束的意义

## 问题

> 为什么需要时序约束？不写约束会发生什么？FPGA 设计中至少需要哪些约束？

> 难度: ★ | 频率: 中频 | 层级: L1

## 结论

时序约束告诉 EDA 工具设计需要跑多快、IO 时序要求是什么。没有约束 = 工具不知道时钟频率 → 按任意方式优化 → 功能错误。

## 三大必要约束

1. **时钟约束** `create_clock` — 告诉工具时钟频率/占空比
2. **IO 约束** `set_input_delay` / `set_output_delay` — 告诉工具外部时序
3. **例外约束** `set_false_path` / `set_multicycle_path` — 告诉工具哪些路径不需要单周期优化

## 无约束的后果

| 场景 | 后果 |
|------|------|
| 不加时钟约束 | 工具可能不优化或过度优化 |
| 不加 IO 约束 | 外部芯片时序违例 → 接口错误 |
| 不加例外约束 | 工具在异步/CDC路径上浪费优化时间 |

## 工程底线

FPGA 上板前至少需要：时钟约束 + IO 约束 = 才能保证基本功能正确。
